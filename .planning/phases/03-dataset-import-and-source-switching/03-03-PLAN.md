# Plan 03-03 - Header Source Switching Experience

## Goal

Conectar a troca de origem de dados ao header e garantir que todo o dashboard reaja corretamente a importacao e selecao da fonte ativa.

## Requirements

- DATA-03
- DATA-04

## Depends on

- 03-01
- 03-02

## Proposed files

- `components/dashboard/header.tsx`
- `components/dashboard/index.tsx`
- `hooks/use-radar-dashboard.ts`
- `components/dashboard/search-panel.tsx`
- `components/dashboard/config-panel.tsx`

## Tasks

1. Transformar a area `Origem ativa` do header em um dropdown/popover funcional.
2. Exibir no dropdown:
   - origem atual
   - outras origens disponiveis
   - metadata curta por origem
   - CTA para importar nova planilha
3. Integrar um fluxo compacto de importacao na UI, com feedback de sucesso/erro.
4. Conectar a troca de origem ao hook principal e garantir refresh coerente dos dados derivados.
5. Reconciliar estado dependente da origem, como termo selecionado e contexto de consulta.

## Done when

- O usuario consegue trocar a origem ativa diretamente do header.
- O usuario consegue iniciar a importacao sem sair do dashboard.
- KPIs, charts, tabela, consulta e configuracao respondem corretamente a mudanca de origem.
- A experiencia permanece harmoniosa com o layout existente.

## Verification

- `pnpm exec tsc --noEmit`
- `pnpm build`
- Teste manual alternando entre origem embarcada e origem importada
