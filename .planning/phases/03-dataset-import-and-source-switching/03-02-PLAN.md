# Plan 03-02 - Versioned Source Registration

## Goal

Transformar um resultado de importacao valido em uma nova origem versionada do RADAR, com metadata consistente e regras seguras de ativacao.

## Requirements

- DATA-01
- DATA-02

## Depends on

- 03-01

## Proposed files

- `lib/radar-data-sources.ts`
- `lib/radar-persistence.ts`
- `lib/radar-schemas.ts`
- `hooks/use-radar-dashboard.ts`

## Tasks

1. Ajustar a criacao da origem importada para gerar versionamento e labels legiveis.
2. Garantir metadata minima consistente:
   - nome da origem
   - nome do arquivo
   - timestamp
   - quantidade de termos
3. Definir regra de ativacao segura apos importacao:
   - ativar imediatamente com rollback simples em caso de erro, ou
   - registrar e ativar por acao explicita subsequente
4. Garantir que o persistence state mantenha apenas uma origem ativa por vez.
5. Expor no hook principal uma acao completa `importDataSource` que valida, registra e devolve resultado para a UI.

## Done when

- Uma importacao valida cria nova origem persistida e identificavel.
- O registry permanece coerente com uma unica origem ativa.
- A UI pode consumir um resultado pronto para feedback ao usuario.

## Verification

- `pnpm exec tsc --noEmit`
- Confirmar que uma nova origem aparece no persistence state apos importacao valida
