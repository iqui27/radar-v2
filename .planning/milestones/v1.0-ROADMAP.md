# Milestone v1.0: RADAR v2 Foundation

**Status:** ✅ SHIPPED 2026-03-31
**Phases:** 1-3
**Total Plans:** 9

## Overview

Esta milestone transformou o RADAR v2 em um dashboard operacional com memoria local, historico de busca e configuracao, e multiplas origens de dados trocaveis a partir do header.

## Phases

### Phase 1: Persistence and Data Registry

**Goal**: Introduzir um modelo persistente e validado para configuracoes, historicos e versoes de dados sem quebrar a leitura atual do dashboard.  
**Depends on**: Nothing (first phase)  
**Plans**: 3 plans

Plans:

- [x] 01-01: Definir contratos de persistence para config snapshots, search history e data sources.
- [x] 01-02: Extrair a orquestracao do dashboard para um state container local-first.
- [x] 01-03: Integrar o registry persistido ao pipeline de enriquecimento do dashboard.

### Phase 2: Search and Config History UX

**Goal**: Mostrar historico util de buscas, snapshots de configuracao e deltas de metricas no fluxo de consulta.  
**Depends on**: Phase 1  
**Plans**: 3 plans

Plans:

- [x] 02-01: Construir componentes de historico de busca e selecao de termos.
- [x] 02-02: Construir timeline de snapshots de configuracao com restore.
- [x] 02-03: Exibir deltas de metricas por termo usando snapshots historicos.

### Phase 3: Dataset Import and Source Switching

**Goal**: Permitir importacao de planilhas como novas versoes de dados e alternar a origem ativa em todo o dashboard.  
**Depends on**: Phase 1  
**Plans**: 3 plans

Plans:

- [x] 03-01: Implementar parser e validacao da planilha importada.
- [x] 03-02: Registrar novas versoes e metadata da origem de dados.
- [x] 03-03: Conectar dropdown de origem a todo o dashboard.

## Milestone Summary

**Key Accomplishments**

1. Base local-first de persistencia para configuracoes, snapshots e registry de origem.
2. Historico recente de busca e restore de contexto diretamente na Consulta.
3. Historico restauravel de configuracoes com snapshots reais no painel de Configuração.
4. Deltas historicos de score, posição, CTR, cliques e impressões no card de análise.
5. Importacao CSV-first de novas bases com validacao e feedback.
6. Dropdown de origem ativa no header com recálculo global do dashboard.

**Key Decisions**

- Persistencia e versionamento ficaram client-side nesta milestone.
- Historico de métricas usa snapshots reais persistidos em vez de derivação ad hoc.
- Importação inicial foi fechada como CSV-first.
- Header virou a superfície única para fonte ativa e importação.

**Issues Deferred**

- Suporte nativo a `.xlsx`.
- Comparações temporais mais profundas sobre histórico de datasets.
